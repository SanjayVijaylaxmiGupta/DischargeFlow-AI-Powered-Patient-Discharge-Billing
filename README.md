# DischargeFlow: AI-Powered Patient Discharge & Billing Orchestration

> UiPath AgentHack — Track 2: Maestro BPMN

## 🎯 Project Description

### The Business Problem
Patient discharge billing in hospitals is currently a fragmented, manual process involving up to 6 different software systems. When a doctor signs a discharge order, clinical and billing teams must manually:
1. Document and write clinical discharge summaries (taking 45-60 minutes of clinical nurse time).
2. Log into legacy insurance portals to verify eligibility and coverage limits (30-60 minutes).
3. Cross-reference procedure codes, apply insurance adjustments, and generate bills (30-60 minutes).
4. Mail paper invoices to patients, creating a slow 30-45 day payment collection cycle.
5. Manually review, call, and negotiate disputed claims when insurance denys coverage (taking 2-4 weeks).

The Cost of Manual Billing:
* High Error Rates (12-15%): Typographical errors on diagnosis or billing codes lead to immediate claim rejections.
* Delayed Cash Flow: Mailing paper bills delays payments by weeks.
* Administrative Burnout: Clinicians waste hours on paperwork instead of patient care.\

### The Solution: DischargeFlow
DischargeFlow is an end-to-end orchestration that automates the entire patient discharge-to-payment pipeline down to **under 2 hours**. By utilizing a single Maestro BPMN process, we coordinate autonomous AI agents, legacy system RPA robots, and human-in-the-loop exception handling. 

* Clinical Summary Generation: Auto-generated by AI in 30 seconds.
* Insurance Eligibility Checks: Auto-checked by RPA via legacy portal screens.
* Billing & Co-pay Calculations: Processed instantly by rules-based robots.
* Digital Invoicing: Secured payment link sent directly via email.
* Intelligent Dispute Resolution: AI Triage Agent pre-analyzes disputes to recommend actions, which human reviewers approve/deny in a single click.

## 🤖 Agent Type Statement

This solution utilizes a **Combination of Coded Agents and Low-Code Agents**:
* Low-Code Agents: The agent configs, system prompts, and tool interfaces were designed and configured directly using the UiPath Studio Web Agent Builder interface.
* Coded Agents: Custom prompts, variable mappings, and rules-based logic were structured inside local repository templates (stored in the `/agents` folder) to allow version control, systematic testing, and coding assistant integration.

## 🧩 UiPath Components Used

Our orchestration uses the following UiPath components:

| Component | Purpose | Type |
|---|---|---|
| Maestro BPMN | Coordinates the sequence of tasks, gateways, and variables across the entire pipeline. | BPMN 2.0 Process |
| ClinicalSummaryAgent | LLM-driven agent that reads diagnoses and clinical notes to generate structured summaries. | AI Agent (Studio Web) |
| ExceptionTriageAgent | LLM-driven agent that pre-analyzes disputed claims to suggest resolutions. | AI Agent (Studio Web) |
| PatientDataLoader | Reads patient database records dynamically using key inputs. | RPA Workflow |
| InsuranceEligibilityCheck | Automates legacy system interface checks to find active coverage limits. | RPA Workflow |
| BillGeneration | Calculates procedure pricing, copays, and remaining patient balances. | RPA Workflow |
| PaymentLinkGenerator | Generates digital invoice links and flags disputes over a $10,000 threshold. | RPA Workflow |
| APClaimReview | Triggers an interactive dialog for human AP review using AI recommendation outputs. | RPA Workflow |

## 🛠️ Setup & Running Instructions

### Prerequisites
* Active access to a UiPath Automation Cloud tenant (such as a labs tenant).
* UiPath Studio (desktop version) for publishing RPA workflows.
* A web browser to access Studio Web and Maestro.

### Option A: Import Solution Package (Recommended)
We have packaged the entire project into a single solution file for easy deployment:
1. Log in to your UiPath Automation Cloud.
2. Go to Solutions -> Import.
3. Import the `.uis` file from the `/solution-package` folder in this repository.
4. Click Deploy. This will create the BPMN process, agents, and RPA packages automatically in your workspace.

### Option B: Manual Setup
1. **RPA Workflows: Open each folder in `/rpa-workflows` using UiPath Studio (desktop) and publish them to your tenant's Shared package feed.
2. **AI Agents: In **Studio Web, create two new AI Agents. Import settings and prompts from `agents/clinical-summary-agent/agent-config.json` and `agents/exception-triage-agent/agent-config.json` respectively. Publish both.
3. BPMN Import: In Maestro, create a new process and import the BPMN XML file from `bpmn/patient-discharge-billing.bpmn`. Re-link the task blocks to your published workflows and agents, mapping variables as defined in the canvas properties.

## 📊 How to Run and Judge the Demo

We have created built-in test profiles in the system. Run the process in Maestro or Orchestrator and try these cases:

### Test Case 1: Clean Happy Path (Fully Automated)
1. Set the process input variable `patientId` to: `PAT-2024-001`
2. Execution: The process loads data for *John Smith*, generates a clinical summary, checks insurance (finds active policy with 80% coverage), calculates a bill, and sends a $9,000 payment link.
3. Outcome: Since the amount is under the $10,000 threshold, the gateway routes directly to Close Record. The run completes automatically with no human steps.

### Test Case 2: Dispute Path (AI + Human Collaboration)
1. Set the process input variable `patientId` to: `PAT-2024-002`
2. Execution: The process loads data for *Sarah Martinez*, checks insurance (returns denied/0% coverage), and generates a $35,000 bill. 
3. AI Triage: Because the patient responsibility is over $10,000, the gateway routes to ExceptionTriageAgent. The AI analyzes the dispute and recommends NEGOTIATE at 65% confidenc due to an emergency exception.
4. Human Review:** An AP Claim Review dialog box will pop up. Review the patient details and the AI advice, type `approved` (or `denied`) in the prompt, and click OK.
5. Outcome: The gateway routes based on your input to close or deny the claim.
